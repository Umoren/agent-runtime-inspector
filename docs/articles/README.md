# Articles

These articles capture the product thinking behind Agent Runtime Inspector.

They are included in the repo so builders can understand the larger architecture ARI is moving toward: context provenance, permission-aware retrieval, tool execution, model routing, and local audit trails for agent runs.

## Articles

- [The AI Integration Layer Is Becoming the New Backend](ai-integration-layer-new-backend/index.md)
- [Permission-Aware Retrieval for Enterprise AI Agents](permission-aware-retrieval-enterprise-ai-agents/index.md)

## Why These Belong Here

ARI sits inside the architecture described by these pieces:

- the AI integration layer around agents, tools, models, and business systems
- the permission-aware retrieval boundary before context reaches an LLM
- the provenance record that shows what an agent knew, what it did, and what changed
